A deliberate scope decision. OpenShift AI is a deep, fast-moving product. A shallow multi-guide series risks appearing to overstate operational depth. This guide takes a different approach: one authoritative architectural reference that demonstrates the thinking a consulting architect brings — component architecture, use case mapping, infrastructure prerequisites, integration reasoning, and the questions that prevent failed engagements. The GPU infrastructure decision traces directly back to VIRT-00 Section 04, where the DL385 Gen11 node selection explicitly reserved PCIe 5.0 ×16 slots for future GPU expansion. This guide is where that forward-planning decision pays off.

Scope note. This guide covers RHOAI architecture, component design, GPU infrastructure, use case reasoning, platform integration, and the consulting practice of scoping an AI engagement. It does not cover step-by-step operator installation, DataScienceCluster CR configuration, Workbench walkthroughs, or pipeline SDK code. Those operational details are intentionally excluded — the guide is an architectural decision record, not an installation runbook. Readers who want to proceed from this reference to a hands-on implementation should refer to the Red Hat OpenShift AI documentation.
