ESXi — VMware's bare-metal hypervisor
ESXi (Elastic Sky X Integrated) is VMware's Type 1 hypervisor — it installs directly onto physical hardware and runs instead of a traditional operating system. There is no Windows or Linux underneath it. ESXi takes full control of the server's CPU, memory, storage, and network, then partitions those resources into virtual machines. In NovaCrestBank's data center, ESXi runs on all five HP ProLiant DL380 Gen11 hosts and will later host Domain Controllers, the Certificate Authority servers, vCenter itself, and the OpenShift node VMs.

What this guide does NOT cover

This guide installs ESXi and performs first-time network configuration using the DCUI (Direct Console User Interface — the blue text menu on the server screen). It does NOT configure vMotion VMkernel adapters, vSAN VMkernel adapters, or NFS VMkernel adapters — those require vCenter and are configured in the vDS guides. After this guide, each host has exactly one thing working: its management network (vmk0 on VLAN 10) and SSH access. That is enough for vCenter to discover and manage the hosts.

Why all 5 hosts are installed before vCenter is deployed
vCenter is itself a VM — it runs as a virtual appliance on ESXi. VMware recommends deploying vCenter before configuring additional ESXi hosts, because vCenter is what manages them. However, the more practical approach for NovaCrestBank is to install ESXi on all five hosts first, verify basic connectivity, and then deploy vCenter — because vCenter's deployment wizard requires at least one ESXi host to deploy the VCSA onto, and having all hosts ready means you can configure the cluster immediately after vCenter comes up.

The DNS chicken-and-egg situation
ESXi installation requires a hostname but not necessarily a working DNS server at install time. NovaCrestBank's DNS lives on the Windows Domain Controllers — which are VMs — which do not exist yet. ESXi can be installed and will work perfectly with just a static IP and a hostname. DNS will be filled in after the Domain Controllers are built. For now, use the Windows DC IP (10.70.0.10) as the DNS server setting — even though it does not yet respond. ESXi will simply get a DNS timeout for resolution attempts but will function normally for everything that matters during the initial installation.

NIC mapping — which vmnic does what
Understanding the physical NIC to vmnic mapping before installation — this determines which NIC carries which traffic
The most important thing to understand before ESXi installation
When ESXi boots, it names each physical NIC as vmnic0, vmnic1, vmnic2, etc. in the order it discovers them. On the HP ProLiant DL380 Gen11, this order is determined by the PCIe slot and port numbering. NovaCrestBank's Cisco switch ports are cabled to specific NIC ports. You must know which vmnic corresponds to which physical port so you configure the right NIC for management traffic — otherwise the management VMkernel ends up on the wrong NIC and the host is unreachable from the network.

HP ProLiant DL380 Gen11 NIC discovery order

The DL380 Gen11 in NovaCrestBank is equipped with the HPE NS204i-u NIC — a six-port 25G SFP28 adapter. ESXi discovers NICs in order of PCIe bus address. On the DL380 Gen11 the order is consistent and predictable:

vmnic #Physical port label on NICConnected to (esxi01)Traffic purposeVLAN
vmnic0Port 1 (leftmost)leaf01 Eth1/1 → Po101Management (primary)10, 70
vmnic1Port 2leaf02 Eth1/1 → Po101Management (secondary — vPC)10, 70
vmnic2Port 3leaf01 Eth1/3 → Po102VM traffic + OCP (primary)20, 50, 60
vmnic3Port 4leaf02 Eth1/3 → Po102VM traffic + OCP (secondary)20, 50, 60
vmnic4Port 5leaf01 Eth1/5 → Po103vSAN + NFS (primary)25, 30
vmnic5Port 6 (rightmost)leaf02 Eth1/5 → Po103vSAN + NFS (secondary)25, 30
ESXi management uses vmnic0 only during initial install
During ESXi installation and first-time DCUI configuration, ESXi creates a temporary standard vSwitch0 with vmnic0 as its uplink. Only vmnic0 carries management traffic at this stage. The other five NICs are physically connected to the switch but ESXi does not yet use them — they sit idle until vCenter and the vDS are configured. This is correct behaviour. Do not worry that only vmnic0 is active after installation.

How to verify the vmnic-to-port mapping during installation

ESXi's installer shows you each network adapter it discovers, including its MAC address. The MAC address is printed on a label on the physical NIC or readable in iLO under Information → System Information → Network Adapters. Cross-referencing the MAC address with the port number confirms you are configuring the right adapter. In practice, the DL380 Gen11 is consistent — vmnic0 is always Port 1 — but verifying with the MAC address is the professional approach.

Preparing the ESXi ISO
Downloading the correct version and understanding why we use a custom HP image — not the generic VMware ISO
Use the HPE Custom ESXi Image — not the generic VMware ESXi ISO
VMware and HP produce a customised ESXi image specifically for HP ProLiant hardware. This custom ISO includes HP-specific drivers (for the storage controller, NIC, and iLO integration), the HP SMT (Smart Array Management) tools, and pre-configured health monitoring for HP hardware components. If you install the generic VMware ESXi ISO on a DL380 Gen11, some HP-specific hardware may not function correctly — the Smart Array controller may not present storage correctly, and iLO health integration may be degraded. Always use the HPE-customised image for HP hardware.

Where to download the HPE Custom ESXi Image

  • 1
    Go to the VMware Customer Connect portal

    Open your browser and go to customerconnect.vmware.com. Log in with your VMware account. If NovaCrestBank has a VMware licence, the account tied to that licence is what you use.

  • 2
    Navigate to Products and Downloads → VMware vSphere → Drivers and Tools

    In the top menu click Products → VMware vSphere. On the vSphere product page click Drivers and Tools. In the search filter type HPE. Look for the entry titled VMware ESXi 8.0 U3 for HPE ProLiant or similar. The filename will look like:
    VMware-ESXi-8.0U3-HPE-800.24195311.iso

  • 3
    Download the ISO to your laptop

    Click the download link. The file is approximately 400–500 MB (larger than the generic ISO because of additional HPE drivers). Save it to a location you can easily find — for example, C:\ISO\VMware-ESXi-8.0U3-HPE.iso on Windows or ~/Downloads/VMware-ESXi-8.0U3-HPE.iso on Mac. You do not need to burn this to a USB drive — iLO mounts it virtually over the network.

  • 4
    Verify the MD5 or SHA256 checksum

    VMware provides a checksum on the download page. Verify your downloaded ISO matches it. On Windows: open PowerShell and run Get-FileHash C:\ISO\VMware-ESXi-8.0U3-HPE.iso -Algorithm SHA256. On Mac: run shasum -a 256 ~/Downloads/VMware-ESXi-8.0U3-HPE.iso. A corrupted ISO causes mysterious install failures — verifying the checksum takes 30 seconds and eliminates that possibility.

Choosing where ESXi installs — the most important decision in the installer
Critical — selecting the correct disk is essential
The DL380 Gen11 in NovaCrestBank has multiple storage devices: two M.2 NVMe SSDs (configured as RAID-1 by the HP Smart Array controller — presented as a single logical drive to ESXi), and potentially other storage visible to the HP Smart Array. ESXi must be installed on the M.2 RAID-1 logical volume — NOT on a vSAN disk, NOT on a shared SAN LUN. Installing ESXi on the wrong disk will either overwrite important data or result in ESXi not booting after the server reboots.

What happens during installation — step by step

The progress bar moves through these operations in sequence. You do not need to do anything — just watch:

  1. Partitioning disk — ESXi creates its partition layout on the M.2 RAID volume. ESXi uses a specific partition structure: a small EFI partition, a boot bank, a second boot bank (for upgrade rollback), a scratch partition, and a large VMFS datastore for VMs.
  2. Installing VMkernel — The ESXi kernel (VMkernel) is written to the boot partitions. This is the core of ESXi.
  3. Installing system files — ESXi's management agents, configuration tools, and the HPE-specific drivers from the custom image are written to disk.
  4. Configuring boot loader — The disk's boot record is configured to load ESXi on startup.

Total installation time: approximately 5–10 minutes when streaming from the iLO virtual media. It is slower than a local USB install because the data transfers over the network from your laptop. Do not close the iLO browser tab during this process.

Step 2: VLAN — set VLAN ID 10

Navigate to VLAN (optional) and press Enter. A text input box appears asking for a VLAN ID. Type 10 and press Enter.

Why VLAN 10 must be set here — this is the most common ESXi installation mistake
The Cisco leaf switch port (Po101 on leaf01) is configured as a trunk port — not an access port. This means the switch expects frames arriving from ESXi to carry an 802.1Q VLAN tag. If you do not set VLAN 10 in the DCUI, ESXi sends untagged frames, which arrive at the switch on the native VLAN (VLAN 1) — not VLAN 10. The switch does not deliver these to the VLAN 10 subnet (10.10.0.0/24), so the host is unreachable even though the cable is plugged in and the link light is on. Setting VLAN 10 in the DCUI causes ESXi to tag all management traffic with VLAN 10, which the switch correctly routes to the 10.10.0.1 gateway.
✓After setting VLAN 10, the VLAN line in the Configure Management Network menu now shows "10" instead of "None". Proceed to IPv4 Configuration to set the static IP address.

SSH on ESXi — enable it but understand the security trade-off
ESXi SSH is disabled by default for security — ESXi's attack surface is minimised when SSH is off. In a production environment, SSH should only be enabled when actively needed and disabled afterwards. For NovaCrestBank's initial build phase, SSH is enabled on all hosts because it is needed for vSphere setup, vSAN configuration, and troubleshooting.
