8. Understanding Terraform Modules

8.1 Reader Doubt Clarification

Question:

What exactly is a Terraform module?

Why do enterprises use modules?

8.2 Real Enterprise Explanation

A Terraform module is a reusable infrastructure component.

8.3 Real Enterprise Example

Instead of manually creating NSX segments repeatedly, enterprises create reusable modules. 

module "banking-prod-segment" {

  source = "./modules/nsx-segment"

  segment_name = "seg-banking-prod"
  gateway       = "10.240.50.1/24"

}

8.4 Understanding Every Directive

Directive Purpose
module Declares reusable infrastructure module
source Module location
segment_name Input variable
gateway NSX segment gateway

9.1 What is a Provider

9.2 Real Enterprise Providers

provider "vsphere" {

  user                 = "svc-terraform@corp.novacrestbank.internal"
  password             = var.vsphere_password
  vsphere_server       = "vcsa01.corp.novacrestbank.internal"

  allow_unverified_ssl = true

}

10. Understanding Terraform State

10.1 Reader Doubt Clarification

Question:

What is Terraform state?

Why do enterprises protect it carefully?

10.2 Real Enterprise Explanation

Terraform state tracks deployed infrastructure.

Without state:

  • Terraform cannot track resources
  • Infrastructure drift occurs
  • Automation becomes dangerous

10.3 Enterprise State Protection

Security Requirement Reason
Encrypted state Contains secrets
Remote backend Shared team access
RBAC controls Prevent unauthorized changes
