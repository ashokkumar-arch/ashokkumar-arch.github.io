1. Overview

In the previous guide, we deployed enterprise workload domains and NSX overlay networking infrastructure.

However, Kubernetes itself still does not exist yet.

A reader may immediately get a very important question:

Question:

We already built:

• ESXi clusters
• NSX networking
• Workload domains
• Enterprise PKI
• Enterprise identity integration

So:

• What exactly creates Kubernetes?
• Where does Kubernetes actually run?
• What is Supervisor?
• What is VKS?
• Is VKS the same as upstream Kubernetes?
• Why do enterprises use this instead of manually installing Kubernetes?

1.1 Real Enterprise Explanation

VMware Cloud Foundation provides enterprise Kubernetes through:

  • vSphere Supervisor
  • VMware Kubernetes Service (VKS)
  • NSX integrated networking
  • vSphere integrated storage
  • Enterprise lifecycle management

This allows enterprises to operate Kubernetes as part of the VMware private cloud platform instead of running disconnected standalone Kubernetes clusters.

2. What is vSphere Supervisor?

2.1 Simple Understanding

vSphere Supervisor converts a vSphere cluster into a Kubernetes-enabled platform. 

+------------------------------------------------------+
|                 vSphere Cluster                      |
|------------------------------------------------------|
| ESXi Hosts                                           |
| vSAN Storage                                         |
| NSX Networking                                       |
+------------------------------------------------------+

                    ENABLE SUPERVISOR

                           ↓

+------------------------------------------------------+
|              Supervisor Cluster                      |
|------------------------------------------------------|
| Kubernetes Control Plane                             |
| Kubernetes APIs                                      |
| Namespace Services                                   |
| Tanzu Integration                                    |
+------------------------------------------------------+

2.2 What Actually Happens Internally?

A reader may get another question:

Question:

When we enable Supervisor:

• What exactly gets deployed?
• Are Kubernetes VMs created?
• Where are they deployed?
• Does Supervisor run directly on ESXi?

2.3 Real Technical Explanation

When Supervisor is enabled:

  • Supervisor control plane virtual machines are automatically deployed
  • Kubernetes APIs become integrated into vSphere
  • Namespaces become Kubernetes-aware
  • NSX networking integration becomes active
  • Kubernetes operators become integrated into the platform

3. What is VMware Kubernetes Service (VKS)?

3.1 Real Enterprise Explanation

VKS is VMware's enterprise Kubernetes service for deploying production Kubernetes clusters on VMware Cloud Foundation.

  • Integrated monitoring and governance

8. Enterprise Storage Policies

8.1 Why Storage Policies Matter

Kubernetes applications require persistent storage.

Enterprise applications cannot rely only on temporary container storage.

8.2 Enterprise Storage Classes

bronze-vsan-policy Development workloads

8.3 Example Kubernetes StorageClass

apiVersion: storage.k8s.io/v1
kind: StorageClass

  name: gold-vsan-storageclass

provisioner: csi.vsphere.vmware.com

parameters:
  storagepolicyname: gold-vsan-policy
