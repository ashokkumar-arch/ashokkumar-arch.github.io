6. Enterprise Ingress Controllers

6.1 What is an Ingress Controller

Kubernetes ingress resources alone do nothing.

An ingress controller:

  • Reads ingress definitions
  • Creates routing rules
  • Handles HTTP/HTTPS traffic
  • Routes requests to applications

10. Example Enterprise Ingress Deployment

10.1 Create Application Namespace

kubectl create namespace banking-apps

10.2 Deploy Sample Application

kubectl create deployment customer-portal \
--image=nginx \
-n banking-apps

10.3 Expose Kubernetes Service

kubectl expose deployment customer-portal \
--port=80 \
--target-port=80 \
--type=ClusterIP \
-n banking-apps

10.4 Create Ingress Resource

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: customer-portal-ingress
  namespace: banking-apps
spec:
  ingressClassName: avi-lb
  tls:
  - hosts:
    - banking.apps.novacrestbank.internal
    secretName: banking-tls-secret

  rules:
  - host: banking.apps.novacrestbank.internal
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: customer-portal
            port:
              number: 80

11. Understanding the Ingress YAML

Directive Explanation
kind: Ingress Creates ingress routing rules
ingressClassName Selects ingress controller
tls Defines HTTPS configuration
host DNS hostname
path Application URL path
backend Target Kubernetes service

14.1 What is a Sidecar Proxy

A sidecar proxy is an additional container attached to application pods.

The proxy:

  • Intercepts traffic
  • Encrypts traffic
  • Applies policies
  • Collects telemetry
