3. What is Kubernetes Admission Control

3.1 Reader Doubt Clarification

Question:

What exactly is admission control?

Is it a firewall?

Is it RBAC?

3.2 Simple Explanation

Admission control acts like a security checkpoint for Kubernetes resources.

Before Kubernetes accepts a deployment:

  • The request is intercepted
  • Policies are evaluated
  • The deployment is either allowed or rejected

3.3 Real Enterprise Workflow

kubectl apply deployment.yaml
                |
                |
API Server Receives Request
                |
                |
Admission Controllers Trigger
                |
                |
Policy Engine Evaluates Rules
                |
     --------------------------
     |                        |
Pass                    Fail
     |                        |
Deployment              Deployment
Created                 Rejected

9. Enterprise Policy Examples

9.1 Block Privileged Containers

apiVersion: kyverno.io/v1
kind: ClusterPolicy

metadata:
  name: block-privileged-containers

spec:

  validationFailureAction: enforce

  rules:

  - name: privileged-containers

    match:
      any:
      - resources:
          kinds:
          - Pod

    validate:
      message: "Privileged containers are not allowed."

      pattern:
        spec:
          containers:
          - securityContext:
              privileged: false

9.2 What This Policy Actually Does

  • Intercepts pod creation requests
  • Checks securityContext
  • Rejects privileged containers
  • Prevents host-level compromise risks

9.3 Block Containers Running as Root

apiVersion: kyverno.io/v1
kind: ClusterPolicy

metadata:
  name: require-non-root

spec:

  validationFailureAction: enforce

  rules:

  - name: validate-runAsNonRoot

    match:
      any:
      - resources:
          kinds:
          - Pod

    validate:
      message: "Containers must not run as root."

      pattern:
        spec:
          securityContext:
            runAsNonRoot: true

9.4 Restrict Images to Enterprise Harbor

apiVersion: kyverno.io/v1
kind: ClusterPolicy

metadata:
  name: restrict-image-registries

spec:

  validationFailureAction: enforce

  rules:

  - name: validate-registries

    match:
      any:
      - resources:
          kinds:
          - Pod

    validate:
      message: "Only Harbor registry images are allowed."

      pattern:
        spec:
          containers:
          - image: "harbor.corp.novacrestbank.internal/*"

9.5 Why This Matters

Enterprise Security Principle:

Production clusters should NEVER allow unrestricted public container images.

This prevents:

• Malware images
• Untrusted dependencies
• Supply chain attacks
• Compliance violations

12. Enterprise Resource Governance

12.1 Real Enterprise Problem

Without limits, a single workload can consume excessive CPU and memory resources.

12.2 Enterprise Resource Policy

apiVersion: kyverno.io/v1
kind: ClusterPolicy

metadata:
  name: require-resource-limits

spec:

  validationFailureAction: enforce

  rules:

  - name: validate-resources

    match:
      any:
      - resources:
          kinds:
          - Pod

    validate:
      message: "CPU and memory limits are required."

      pattern:
        spec:
          containers:
          - resources:
              limits:
                memory: "?*"
                cpu: "?*"
