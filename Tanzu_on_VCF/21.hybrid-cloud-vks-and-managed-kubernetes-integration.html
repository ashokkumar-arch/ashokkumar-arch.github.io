3. Why We Selected Azure AKS

3.1 Reader Doubt Clarification

Question:

Why AKS?

Why not EKS or GKE?

3.2 Real Enterprise Reasoning

AKS is commonly selected in enterprise environments because:

  • Strong Microsoft enterprise integration
  • Enterprise identity federation support
  • Hybrid networking maturity
  • Azure ExpressRoute integration
  • Strong banking and enterprise adoption
  • Excellent Kubernetes operational maturity

3.3 Important Understanding

This does NOT mean:

  • EKS is bad
  • GKE is bad

The choice depends on enterprise requirements.

15. Enterprise AKS Cluster Deployment Example

az aks create \
  --resource-group rg-ncb-aks-prod \
  --name aks-prod-bank01 \
  --node-count 3 \
  --network-plugin azure \
  --enable-managed-identity \
  --enable-oidc-issuer \
  --generate-ssh-keys

15.1 Understanding Every Directive

Directive Purpose
--resource-group Azure resource container
--name AKS cluster name
--node-count Initial worker node count
--enable-managed-identity Secure Azure identity integration
--enable-oidc-issuer OIDC federation support
