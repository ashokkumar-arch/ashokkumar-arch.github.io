1.1 Real Enterprise Understanding

Real enterprise Kubernetes environments NEVER rely on engineers manually running kubectl apply against production clusters.

Instead, enterprise environments use GitOps.

Git becomes the:

  • Single source of truth
  • Deployment control plane
  • Audit system
  • Rollback mechanism
  • Environment promotion engine

2. What is GitOps

2.1 Simple Beginner Understanding

GitOps means: 

Git Repository
        ↓

Desired Kubernetes State

        ↓

ArgoCD Watches Git

        ↓

ArgoCD Applies Changes

        ↓

Cluster State Matches Git

2.2 Very Important Enterprise Principle

Golden Rule:

In GitOps:

Git is the truth.

Kubernetes clusters are NOT the truth.

9. Enterprise ArgoCD Application Example

9.1 ArgoCD Application Definition

apiVersion: argoproj.io/v1alpha1
kind: Application

metadata:
  name: customer-portal

spec:

  project: production

  source:
    repoURL: https://git.corp.novacrestbank.internal/gitops

    targetRevision: main

    path: applications/customer-portal

  destination:
    server: https://kubernetes.default.svc

    namespace: customer-portal-prod

  syncPolicy:
    automated:
      prune: true
      selfHeal: true

9.3 Directive Explanation

Directive Purpose

selfHeal Corrects manual cluster drift
targetRevision Git branch or tag
path Manifest directory inside repository
