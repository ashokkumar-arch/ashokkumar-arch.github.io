Deploying Harbor Enterprise Registry for VMware Kubernetes Service (VKS), implementing secure enterprise software supply chain governance, vulnerability scanning, image signing, RBAC, replication, air-gapped operations, and production-grade container image lifecycle management.
Harbor Registry Container Security Image Governance VKS Software Supply Chain Air-Gapped Enterprise

1. Overview

We successfully built:

  • VMware Cloud Foundation
  • NSX enterprise networking
  • vSphere Supervisor
  • VKS Kubernetes clusters
  • Enterprise storage integration
  • Enterprise namespace governance

However, applications still cannot be deployed securely yet.

A reader may immediately get a question:

Question:

Kubernetes deploys container images.

So:

• Where are enterprise container images stored?
• Can developers directly pull images from Docker Hub?
• How do enterprises scan images for vulnerabilities?
• How do enterprises prevent malicious images?
• How do air-gapped environments work?
• What if Docker Hub becomes unavailable?
• How do enterprises control which images are allowed?

1.1 Real Enterprise Understanding

Real enterprise environments NEVER allow production Kubernetes clusters to pull images directly from public registries without governance.

Instead, enterprises deploy internal enterprise container registries with:

  • Image governance
  • RBAC
  • Vulnerability scanning
  • Image signing
  • Replication
  • Air-gapped image lifecycle management
  • Audit logging
  • Immutable artifact retention

2. Why Enterprise Container Registry Exists

2.1 Simple Understanding

A container registry is similar to:

  • VMware Content Library for VM templates
  • Git repositories for source code

But specifically for container images.

2.2 Real Enterprise Risk Without Internal Registry

Risk Problem
Docker Hub outage Production deployments fail

Internet dependency Air-gapped environments fail

6.3 Enterprise Harbor values.yaml

expose:
  type: ingress

  tls:
    enabled: true
    certSource: secret

  ingress:
    hosts:
      core: harbor.corp.novacrestbank.internal

externalURL: https://harbor.corp.novacrestbank.internal

harborAdminPassword: "REDACTED"

persistence:
  enabled: true

  persistentVolumeClaim:
    registry:
      storageClass: gold-vsan-storageclass
      size: 500Gi

    jobservice:
      storageClass: gold-vsan-storageclass
      size: 50Gi

trivy:
  enabled: true

notary:
  enabled: true
