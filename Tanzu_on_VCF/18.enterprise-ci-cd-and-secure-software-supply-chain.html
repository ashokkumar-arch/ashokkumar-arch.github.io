3. Why Enterprises Need Secure Software Supply Chains

3.1 Reader Doubt Clarification

Question:

Why do enterprises care so much about container image security?

Why not simply:

docker build
docker push

3.2 Real Enterprise Explanation

Because attackers target software supply chains.

Modern enterprise attacks often happen through:

  • Compromised container images
  • Malicious dependencies
  • Fake artifacts
  • Stolen build credentials
  • Unverified software packages

10. Enterprise Pipeline YAML

10.1 Example Tekton Pipeline

apiVersion: tekton.dev/v1beta1
kind: Pipeline
metadata:
  name: banking-app-pipeline

spec:

  tasks:

  - name: build-image

  - name: security-scan

  - name: sign-image

  - name: push-image

  - name: update-gitops-repo

10.2 Understanding Every Directive

Directive Purpose
apiVersion Tekton API version
kind: Pipeline Creates CI/CD pipeline
metadata.name Pipeline name
spec Pipeline configuration
tasks Pipeline execution stages

12. Enterprise Image Signing with Cosign

12.1 Reader Doubt Clarification

Question:

Why sign container images?

Why not simply trust internal developers?

12.2 Real Enterprise Explanation

Image signing ensures:

  • Artifact authenticity
  • Tamper detection
  • Supply chain trust
  • Compliance validation

12.3 Generate Cosign Keys

cosign generate-key-pair

12.4 Sign Container Image

cosign sign \
harbor.corp.novacrestbank.internal/banking/customer-api:v1

12.5 Verify Image Signature

cosign verify \
harbor.corp.novacrestbank.internal/banking/customer-api:v1

12.6 Example Verification Output

Verification for harbor.corp.novacrestbank.internal/banking/customer-api:v1 --
The following checks were performed:
✓ Signature verified
✓ Claims validated
✓ Transparency log verified
