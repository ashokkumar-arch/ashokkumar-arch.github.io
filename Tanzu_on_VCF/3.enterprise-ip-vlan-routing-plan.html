1. Overview

This guide defines the enterprise networking standards for the NovaCrestBank private cloud modernization platform built on VMware Cloud Foundation, NSX, and Tanzu Kubernetes Service.

The purpose of this guide is to establish a production-grade networking architecture that supports:

  • VMware Cloud Foundation infrastructure
  • NSX overlay networking
  • Enterprise Kubernetes networking
  • Disaster recovery failover
  • Zero-trust segmentation
  • Platform engineering automation
  • Future hybrid-cloud integration
  • Future AI-ready infrastructure expansion
Important:

This guide assumes that the physical data center infrastructure already exists physically, including:

• Racks
• Cabling
• Power redundancy
• Spine-leaf switching fabric hardware
• Enterprise WAN connectivity between primary and DR sites

2. Enterprise Network Design Principles

Design Principle Description
High Availability Eliminate single points of failure across networking layers.
Scalability Support future Kubernetes and AI workload growth.
Segmentation Separate management, storage, Kubernetes, and application traffic.
Automation Ready Enable Infrastructure-as-Code and GitOps operations.
DR Consistency Maintain consistent networking across primary and DR sites.
Hybrid Cloud Ready Prepare for future AKS connectivity and workload federation.

6. BGP Routing Architecture

6.1 Why BGP?

BGP provides scalable dynamic routing between the physical underlay fabric and NSX overlay environment.

Modern enterprise data centers use BGP extensively because it:

  • Supports scalable route distribution
  • Improves convergence
  • Enables ECMP routing
  • Integrates cleanly with NSX
  • Supports future hybrid-cloud connectivity
